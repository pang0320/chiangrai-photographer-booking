คำอธิบายไฟล์ breadcrumb.php และ csrf.php

========================================
1. ไฟล์ includes/breadcrumb.php
========================================
คือไฟล์คอมโพเนนต์ที่ทำหน้าที่สร้าง Breadcrumb Navigation (ป้ายบอกทาง/แถบนำทาง) ให้กับหน้าเว็บไซต์

หน้าที่หลักๆ ของไฟล์นี้ประกอบด้วย:
- ดึงค่า URL ปัจจุบัน: นำที่อยู่ URL ที่ผู้ใช้กำลังเปิดอยู่มาแยกส่วน (เช่น /admin/dashboard.php)
- แปลงชื่อเป็นภาษาไทย: มีตัวแปร $breadcrumbLabels ที่ทำหน้าที่จับคู่ชื่อโฟลเดอร์หรือชื่อไฟล์ ให้เป็นชื่อภาษาไทยที่อ่านง่าย เช่น admin -> 'ผู้ดูแลระบบ', dashboard.php -> 'แดชบอร์ด', bookings.php -> 'คำขอจอง'
- สร้างแถบเมนู (HTML): นำส่วนที่แยกไว้มาต่อกันเป็นลิงก์ เพื่อแสดงผลบนหน้าเว็บ (เช่น หน้าแรก / ผู้ดูแลระบบ / แดชบอร์ด) โดยใช้สไตล์ของ Tailwind CSS เพื่อให้สวยงามและคลิกย้อนกลับไปหน้าก่อนหน้าได้ง่าย

ไฟล์นี้จะถูกนำไปเรียกใช้ (include) ในหน้าอื่นๆ ที่ต้องการให้มีแถบนำทางขึ้นที่ส่วนหัวของหน้าเพจ (โดยจะไม่แสดงผลในหน้าแรกสุดอย่าง index.php)


========================================
2. ไฟล์ includes/csrf.php
========================================
CSRF (Cross-Site Request Forgery) คือการโจมตีประเภทหนึ่งบนเว็บไซต์ที่แฮกเกอร์หลอกให้ผู้ใช้ที่ล็อกอินอยู่แล้ว ทำการส่งคำสั่ง (เช่น โอนเงิน, เปลี่ยนรหัสผ่าน, หรือแก้ไขข้อมูล) ไปยังเว็บไซต์เป้าหมายโดยที่ผู้ใช้ไม่รู้ตัว

ไฟล์ includes/csrf.php ในโปรเจกต์นี้ ถูกสร้างขึ้นมาเพื่อ ป้องกันการโจมตีแบบ CSRF โดยเฉพาะ ซึ่งทำงานเปรียบเสมือนเป็นระบบสร้างและตรวจบัตรคิวลับ ประกอบด้วย 3 ฟังก์ชันหลัก ได้แก่:

- csrf_token(): ทำหน้าที่สร้างรหัสความปลอดภัย (Token) ขึ้นมาแบบสุ่ม แล้วเก็บไว้ในระบบ (Session) ของผู้ใช้คนนั้นๆ
- csrf_field(): ทำหน้าที่สร้างช่องกรอกข้อมูลแบบซ่อน (Hidden Input) เพื่อนำไปฝังไว้ในหน้าฟอร์มต่างๆ (เช่น ฟอร์มสมัครสมาชิก, ฟอร์มล็อกอิน, หรือฟอร์มบันทึกข้อมูล)
- verify_csrf(): เมื่อผู้ใช้กดปุ่มบันทึกหรือส่งฟอร์ม (POST Request) ฟังก์ชันนี้จะทำหน้าที่ตรวจเช็คว่า รหัส Token ที่แนบมากับฟอร์มนั้น ตรงกับ รหัส Token ที่เก็บไว้ในระบบของผู้ใช้หรือไม่
  * ถ้าตรงกัน: อนุญาตให้ทำรายการต่อ เพราะมั่นใจได้ว่าคำสั่งถูกส่งมาจากหน้าเว็บของเราจริงๆ
  * ถ้าไม่ตรงกัน หรือไม่มีส่งมา: จะถูกบล็อก (แสดงข้อความ 'Invalid CSRF token' พร้อมรหัส 419) เพราะอาจจะถูกส่งมาจากเว็บของแฮกเกอร์

สรุปสั้นๆ คือ มันเป็นกลไกที่ช่วยยืนยันว่าข้อมูลที่ส่งเข้ามาในระบบนั้น มาจากฟอร์มในเว็บไซต์ของเราจริงๆ ไม่ได้ถูกสั่งการโดยเว็บไซต์อื่น
