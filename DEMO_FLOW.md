# Demo Flow แบบละเอียด: Chiang Rai Photographer Booking Platform

ไฟล์นี้ใช้เป็นสคริปต์พรีเซนต์ระบบแบบละเอียด เปิดอ่านตามได้เลย  
แนะนำให้เปิดเว็บที่ `http://localhost:8000`

## 0. เตรียมก่อน Demo

### 0.1 บัญชีสำหรับ Demo

รหัสผ่านทุกบัญชีคือ `password`

| Role | Email | ใช้ทำอะไร |
| --- | --- | --- |
| Admin | `admin@example.com` | อนุมัติช่างภาพ จัดการข้อมูล ดูรายงาน ดู activity logs |
| Customer | `customer@example.com` | ค้นหา จอง รีวิว รายงานปัญหา ดูประวัติ |
| Photographer | `northstudio@example.com` | จัดการโปรไฟล์ วันว่าง พื้นที่รับงาน ประเภทงาน คำขอจอง บทความ |

### 0.2 URL ที่ควรเปิดเตรียมไว้

เปิด browser tab ไว้ 4 กลุ่ม จะพรีเซนต์ลื่นขึ้น:

1. Public:
   - `http://localhost:8000/index.php`
   - `http://localhost:8000/photographers.php`
   - `http://localhost:8000/blog.php`
   - `http://localhost:8000/faq.php`

2. Customer:
   - `http://localhost:8000/login.php`
   - หลัง login: `http://localhost:8000/customer/dashboard.php`

3. Photographer:
   - `http://localhost:8000/login.php`
   - หลัง login: `http://localhost:8000/photographer/dashboard.php`

4. Admin:
   - `http://localhost:8000/login.php`
   - หลัง login: `http://localhost:8000/admin/dashboard.php`

### 0.3 จุดย้ำที่ต้องพูดตั้งแต่ต้น

พูดสั้น ๆ ก่อนเริ่ม demo:

> ระบบนี้เป็นแพลตฟอร์มค้นหาและจองช่างภาพในจังหวัดเชียงราย มี 3 role คือ ลูกค้า ช่างภาพ และผู้ดูแลระบบ เว็บไซต์ไม่มีระบบชำระเงิน ไม่มีตะกร้าสินค้า และไม่มี payment gateway ทำหน้าที่เป็นตัวกลางให้ลูกค้าค้นหา ส่งคำขอจอง ติดต่อ และรีวิวหลังงานเสร็จเท่านั้น ส่วนราคาและการชำระเงินลูกค้ากับช่างภาพตกลงกันเองภายนอกระบบ

### 0.4 สิ่งที่ควรเช็คก่อนเริ่ม

1. เว็บเปิดได้ที่ `http://localhost:8000`
2. Database มี seed data แล้ว เช่น ช่างภาพ หมวดหมู่ อำเภอ booking review article
3. Login demo ทั้ง 3 role ได้
4. หน้าแรกมีรูป local แสดง ไม่พึ่ง internet
5. ปุ่มเมนูบน navbar มีชื่อผู้ login และ role แสดง
6. ถ้าใช้ Docker ให้เช็ค container:
   - PHP web: port `8000`
   - MariaDB: port ตาม `docker.yml`
   - phpMyAdmin: port ตาม `docker.yml`

### 0.5 ถ้าเจอปัญหาระหว่าง Demo

| ปัญหา | วิธีแก้ตอนพรีเซนต์ |
| --- | --- |
| Login ไม่ได้ | บอกว่ารหัส demo คือ `password` แล้วเช็คว่า import `database.sql` แล้ว |
| ไม่มีข้อมูลในหน้า | ไปหน้า admin หรือใช้ seed data ใหม่ |
| รูปไม่ขึ้น | บอกว่าระบบรองรับ local upload และ placeholder local แล้ว จากนั้น refresh |
| จองไม่ได้ | เลือกวันที่ที่ช่างภาพตั้งเป็น `ว่าง` |
| รีวิวไม่ได้ | ต้องเลือก booking ที่สถานะ `completed` และยังไม่เคยรีวิว |
| รายการน้อย | อธิบายว่าทุก block แสดงไม่เกิน 5 รายการ และมี pagination เฉพาะ block |

# 1. Demo Flow ผู้เยี่ยมชม

เป้าหมาย: แสดงว่าคนทั่วไปเข้าดูข้อมูลได้โดยไม่ต้องสมัคร แต่การจอง รีวิว รายการโปรด และรายงานปัญหาต้อง login

## 1.1 เข้าเว็บหน้าแรก

URL: `/index.php`

### ขั้นตอนที่ทำ

1. เปิดหน้าแรก
2. ชี้ให้ดู navbar:
   - หน้าแรก
   - ค้นหาช่างภาพ
   - บทความ
   - คำถามที่พบบ่อย
   - ติดต่อผู้ดูแลระบบ
   - Login/Register
3. ชี้ให้ดู hero section:
   - search box
   - สถิติ
   - หมวดหมู่
   - ช่างภาพแนะนำ
   - ตัวอย่างงานถ่ายภาพ
   - รีวิวลูกค้า
   - FAQ
   - CTA สมัครเป็นช่างภาพ

### สิ่งที่ต้องพูด

> หน้าแรกออกแบบเป็น marketplace ไม่ใช่เว็บฟอร์มธรรมดา ผู้เยี่ยมชมสามารถดูข้อมูลสาธารณะได้ก่อน เช่น ช่างภาพ หมวดหมู่ ตัวอย่างงาน รีวิว บทความ และ FAQ โดยยังไม่ต้องสมัคร

### ผลที่ควรเห็น

- หน้าแรกดูเต็ม มีหลาย section
- มี icon Font Awesome
- มี card, shadow, rounded, badge, CTA
- ไม่มีการเด้ง login ถ้าแค่ดูข้อมูล

### ถ้าจารย์ถาม

ถาม: ทำไม guest ดูได้?  
ตอบ: เพราะเป็น public marketplace ต้องให้ผู้ใช้ดูข้อมูลก่อนตัดสินใจสมัคร แต่ action ที่เกี่ยวกับข้อมูลส่วนตัว เช่น จอง รีวิว favorite และ report จะบังคับ login

## 1.2 ค้นหาช่างภาพจากหน้าแรก

URL: `/index.php`

### ขั้นตอนที่ทำ

1. ไปที่ search box บน hero
2. เลือกอำเภอ เช่น `เมืองเชียงราย`
3. เลือกประเภทงาน เช่น `งานแต่งงาน` หรือ `รับปริญญา`
4. เลือกวันที่ต้องการจ้าง
5. กดปุ่มค้นหา
6. ระบบไปหน้า `/photographers.php`

### สิ่งที่ต้องพูด

> การค้นหาตั้งใจให้ไม่เกิน 3 ขั้นตอนหลัก คือ เลือกพื้นที่ เลือกประเภทงาน เลือกวันที่ จากนั้นระบบจะแสดงช่างภาพที่ตรงเงื่อนไข

### ผลที่ควรเห็น

- ไปหน้าค้นหาช่างภาพ
- filter ที่เลือกแสดงผลต่อ
- มี search summary เช่น พบช่างภาพกี่คน

## 1.3 ค้นหาช่างภาพแบบละเอียด

URL: `/photographers.php`

### ขั้นตอนที่ทำ

1. เปิดหน้าค้นหาช่างภาพ
2. ชี้ header banner ด้านบน
3. ชี้ filter sidebar:
   - อำเภอ
   - ประเภทงาน
   - วันที่ต้องการจ้าง
   - คะแนนขั้นต่ำ
   - ราคาเริ่มต้นโดยประมาณ
   - ชื่อช่างภาพ
4. เลือก filter 2-3 อย่าง
5. กดค้นหา
6. เปลี่ยน sort:
   - คะแนนสูงสุด
   - รีวิวมากที่สุด
   - ใหม่ล่าสุด
   - ราคาเริ่มต้นต่ำสุด
7. ชี้ active filter chips
8. ชี้ pagination ถ้ารายการเกิน 5 รายการ

### สิ่งที่ต้องพูด

> หน้านี้ใช้สำหรับค้นหาช่างภาพที่ผ่านการอนุมัติแล้วเท่านั้น ช่างภาพ pending, rejected หรือ suspended จะไม่แสดงบน public

> ราคาที่แสดงเป็นราคาเริ่มต้นโดยประมาณ พร้อมหน่วยบาท เว็บไซต์ไม่ได้รับชำระเงิน ลูกค้าต้องตกลงราคาและจ่ายเงินกับช่างภาพโดยตรง

### ผลที่ควรเห็น

- การ์ดช่างภาพมีรูป ชื่อ พื้นที่ ประเภทงาน คะแนน จำนวนรีวิว ราคา และ badge
- การ์ดกดได้ทั้งการ์ด
- ปุ่มย่อย เช่น ดูโปรไฟล์ จอง รายการโปรด ไม่ชนกับการคลิกการ์ด

### ถ้าจารย์ถาม

ถาม: ทำไมไม่แสดงช่างภาพทุกคน?  
ตอบ: เพื่อความน่าเชื่อถือ ระบบจะแสดงเฉพาะช่างภาพที่ admin อนุมัติแล้ว และยังไม่ถูกระงับ

## 1.4 Demo กรณีไม่พบช่างภาพและแนะนำใกล้เคียง

URL: `/photographers.php`

### ขั้นตอนที่ทำ

1. เลือกอำเภอที่มีโอกาสไม่พบผลลัพธ์
2. เลือกประเภทงานเฉพาะเจาะจง
3. กดค้นหา
4. ถ้าไม่พบ ให้ชี้ section `ช่างภาพใกล้เคียงที่แนะนำ`
5. ชี้ข้อความอธิบายอำเภอที่เลือก
6. ชี้ข้อความอธิบายระยะทางประมาณ

### สิ่งที่ต้องพูด

> ถ้าไม่พบช่างภาพในอำเภอที่เลือก ระบบจะหาอำเภอใกล้เคียงโดยใช้ latitude/longitude ของอำเภอและคำนวณด้วย Haversine Formula ระยะทางที่แสดงเป็นระยะประมาณจากพิกัดอำเภอ ไม่ใช่ระยะทางถนนจริง

> การแนะนำใกล้เคียงยังล็อกตามประเภทงานที่ผู้ใช้เลือก เพื่อให้ผลลัพธ์ไม่หลุดจากความต้องการของลูกค้า

### ผลที่ควรเห็น

- มี empty state สวย ๆ ถ้าไม่พบตรงเงื่อนไข
- มี section แนะนำช่างภาพใกล้เคียง
- มีข้อความชัดว่าเลือกอำเภออะไร
- มีคำอธิบายเรื่องระยะทางโดยประมาณ

### ถ้าจารย์ถาม

ถาม: Haversine คืออะไร?  
ตอบ: เป็นสูตรคำนวณระยะห่างระหว่างจุด 2 จุดบนพื้นโลกจาก latitude/longitude เหมาะกับการแนะนำอำเภอใกล้เคียงในระบบค้นหา

- profile view เพิ่มเมื่อเปิดหน้า
- ถ้า login แล้วจะบันทึก recently viewed

# 2. Demo Flow ลูกค้า

เป้าหมาย: แสดงตั้งแต่ login, ค้นหา, จอง, ดูสถานะ, รีวิว, รายงานปัญหา และดูประวัติของตัวเอง

## 2.1 Login ลูกค้า

URL: `/login.php`

### ขั้นตอนที่ทำ

1. เปิดหน้า login
2. กรอก:
   - Email: `customer@example.com`
   - Password: `password`
3. กดเข้าสู่ระบบ
4. ระบบ redirect ไป `/customer/dashboard.php`
5. ชี้ navbar ว่าแสดงชื่อผู้ใช้และ role แล้ว

# 7. Demo Flow Security

เป้าหมาย: อธิบาย security แบบไม่ต้องลงเทคนิคยาว แต่ตอบคำถามได้

## 7.1 SQL Injection

พูด:

> Query ทั้งระบบใช้ PDO Prepared Statement ไม่ต่อ string SQL จาก input ตรง ๆ เพื่อลดความเสี่ยง SQL Injection

## 7.2 XSS

พูด:

> ข้อมูลที่แสดงจากฐานข้อมูลผ่าน htmlspecialchars และเนื้อหาบทความที่เป็น rich text มีการจำกัด tag ที่อนุญาต ไม่ปล่อย event handler หรือ script

## 7.3 CSRF

พูด:

> ฟอร์มสำคัญ เช่น login, register, booking, review, profile update และ delete action มี CSRF token

## 7.4 File Upload Risk

พูด:

> Upload ตรวจ extension, MIME type, getimagesize, จำกัดชนิด jpg/jpeg/png/webp, เปลี่ยนชื่อไฟล์เป็น random hash และมี `.htaccess` ป้องกัน execute PHP ใน uploads

## 7.5 Role-Based Access Control

พูด:

> ทุกหน้าหลังบ้านตรวจ role เช่น customer เข้า admin ไม่ได้ ช่างภาพดู booking ได้เฉพาะของตัวเอง ลูกค้าดู booking ได้เฉพาะของตัวเอง

## 7.6 Activity Log

พูด:

> Action สำคัญถูกบันทึกใน activity_logs เพื่อให้ตรวจสอบย้อนหลังได้

# 9. คำตอบเตรียมไว้ถ้าจารย์ถาม

## ถาม: ระบบมี payment ไหม?

ตอบ:

> ไม่มีครับ ระบบนี้ไม่มี payment gateway ไม่มีตะกร้า ไม่มี invoice และไม่มี commission เว็บไซต์เป็นเพียงตัวกลางค้นหา จอง ติดต่อ และรีวิว ส่วนราคาและการชำระเงินตกลงกันเองภายนอกระบบ

## ถาม: ทำไมต้องให้ admin อนุมัติช่างภาพ?

ตอบ:

> เพื่อคัดกรองข้อมูลก่อนเผยแพร่ ช่างภาพที่ pending, rejected หรือ suspended จะไม่แสดงในหน้า public search

## ถาม: ถ้าไม่พบช่างภาพในอำเภอที่เลือกทำยังไง?

ตอบ:

> ระบบใช้ latitude/longitude ของอำเภอคำนวณระยะใกล้เคียงด้วย Haversine Formula แล้วแนะนำช่างภาพอำเภอใกล้เคียง โดยยังล็อกตามประเภทงานที่เลือก

## ถาม: ลูกค้ารีวิวมั่วได้ไหม?

ตอบ:

> รีวิวได้เฉพาะ booking ของลูกค้าคนนั้นที่สถานะ completed แล้ว และ 1 booking รีวิวได้ 1 ครั้ง

## ถาม: Direct access หน้า admin ได้ไหม?

ตอบ:

> ไม่ได้ครับ หน้าหลังบ้านมี requireLogin และ requireRole ตรวจ role ทุกหน้า

## ถาม: รูป upload ปลอดภัยไหม?

ตอบ:

> Upload ตรวจ extension, MIME type, ขนาดไฟล์, getimagesize และเปลี่ยนชื่อเป็น random hash พร้อมป้องกัน execute PHP ใน uploads

## ถาม: ทำไม activity logs สำคัญ?

ตอบ:

> ใช้ตรวจสอบย้อนหลังว่าใครทำ action อะไรกับข้อมูลไหน เมื่อไหร่ และจาก IP/user agent ไหน ช่วยเรื่อง audit และ debugging

## ถาม: ราคาในเว็บคือราคาจริงไหม?

ตอบ:

> เป็นราคาเริ่มต้นโดยประมาณ มีหน่วยบาทชัดเจน เว็บไซต์ไม่รับชำระเงิน ลูกค้าต้องตกลงราคาสุดท้ายกับช่างภาพโดยตรง

## ถาม: Guest ทำอะไรได้บ้าง?

ตอบ:

> Guest ดูข้อมูล public ได้ เช่น หน้าแรก ค้นหาช่างภาพ โปรไฟล์ บทความ FAQ ติดต่อเว็บ แต่ถ้าจะจอง รีวิว favorite หรือ report ต้อง login

## ถาม: ระบบรองรับมือถือไหม?

ตอบ:

> รองรับครับ UI ทำแบบ mobile first ใช้ responsive layout, card, sidebar/navbar ที่ปรับตามจอ และปุ่ม CTA ชัดเจน
